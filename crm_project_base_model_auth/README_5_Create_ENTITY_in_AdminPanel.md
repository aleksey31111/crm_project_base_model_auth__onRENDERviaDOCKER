Административная панель Django (`/admin`) доступна только для пользователей с правами администратора (is_staff или is_superuser). 
Если у вас нет такого аккаунта, обратитесь к тому, кто установил систему, 
или создайте суперпользователя командой `python manage.py createsuperuser` (это делается один раз при развёртывании). 
Далее предполагается, что вы уже вошли в админку.

## 1. Вход в админку

- Перейдите по адресу `http://127.0.0.1:8000/admin`.
- Введите логин и пароль учётной записи администратора.
- После входа вы увидите панель со всеми зарегистрированными моделями.

## 2. Основные разделы админки

Группы моделей (приложения):
- **Accounts** – пользователи и их профили.
- **Clients** – клиенты.
- **Contracts** – контракты и платежи.
- **Tasks** – задачи и комментарии к ним.
- **Notifications** – уведомления и настройки.
- **Logs** – логи API (если включено).
- **Analytics** – отчёты (если есть).

## 3. Создание пользователя

1. Нажмите **Accounts → Users**.
2. Кнопка **«Добавить пользователя»**.
3. Заполните поля:
   - **Username** – логин (обязательно).
   - **Password** – пароль (будет захеширован).
   - **Email**, **Имя**, **Фамилия**, **Телефон** – по желанию.
   - **Роль** – выберите из списка (ADMIN, MANAGER, SALES, SUPPORT, VIEWER).
   - **Отдел**, **Должность**, **Дата найма** – опционально.
   - Флаги **Активен**, **Персонал**, **Суперпользователь** – ставьте по необходимости.
4. Нажмите **«Сохранить»**.
5. После сохранения появится возможность загрузить **аватар** и заполнить дополнительный 
**профиль** (дата рождения, Telegram, LinkedIn и др.).

## 4. Создание клиента

1. **Clients → Clients** → **«Добавить клиент»**.
2. Заполните обязательные поля:
   - **Тип клиента**: физическое лицо / юридическое лицо / ИП.
   - **Полное наименование** – для юр. лиц или ФИО для физ. лиц.
   - **Статус**: активен / неактивен / потенциальный.
3. Для юр. лиц: ИНН, КПП, ОГРН.
4. Для физ. лиц: фамилия, имя, отчество, дата рождения.
5. Контактная информация: телефон, email, сайт, адрес, город, страна, индекс.
6. **Ответственный менеджер** – выберите пользователя из списка.
7. **Источник** – откуда появился клиент.
8. **Логотип** – можно загрузить картинку.
9. Сохраните.

## 5. Создание контракта

1. **Contracts → Contracts** → **«Добавить контракт»**.
2. Заполните:
   - **Клиент** – выберите из существующих.
   - **Номер договора** – уникальный.
   - **Название**, **Тип**, **Описание**.
   - **Дата начала**, **Дата окончания**.
   - **Сумма** (в рублях).
   - **Оплачено** – можно указать частичную оплату.
   - **Ответственный менеджер**.
   - **Файл договора** – прикрепить PDF/документ.
3. Сохраните.
4. После сохранения можно добавлять **платежи** (вкладка внизу страницы контракта).

## 6. Создание платежа

1. Откройте страницу нужного контракта.
2. Внизу найдите раздел **Платежи** и нажмите **«Добавить ещё один платёж»**.
3. Укажите:
   - **Сумма**.
   - **Дата оплаты**.
   - **Способ оплаты** (наличные, перевод, карта, электронные деньги).
   - **Комментарий** (опционально).
4. Сохраните. Сумма контракта и статус оплаты обновятся автоматически.

## 7. Создание задачи

1. **Tasks → Tasks** → **«Добавить задачу»**.
2. Заполните:
   - **Название**, **Описание**.
   - **Статус**: новая / в работе / завершена / отменена.
   - **Приоритет**: низкий / средний / высокий / срочный.
   - **Срок выполнения** – дата и время.
   - **Исполнитель** – пользователь, ответственный за задачу.
   - **Клиент**, **Контракт** – опционально.
   - **Плановые часы**, **Фактические часы** – для оценки трудозатрат.
3. Сохраните.
4. К задаче можно добавлять **комментарии** 
(также в админке или на странице задачи на сайте).

## 8. Создание уведомления

1. **Notifications → Notifications** → **«Добавить уведомление»**.
2. Выберите **Пользователя**.
3. Укажите **Тип** (информация, успех, предупреждение, ошибка).
4. Заголовок и сообщение.
5. Ссылка (опционально) – куда перейдёт пользователь.
6. Флаг **«Прочитано»** можно выставить вручную.
7. Сохраните. Уведомление появится у пользователя.

## 9. Настройки уведомлений пользователя

- **Notifications → Notification preferences** – здесь можно задать 
глобальные настройки для каждого пользователя (email, какие события отслеживать).

## 10. Просмотр и управление логами API

- **Logs → Api logs** – просмотр запросов к API и вебхукам (если логирование включено).

## 11. Экспорт данных из админки

Почти для каждой модели в админке есть кнопка **«Экспортировать выбранные»** (если действие добавлено). 
Выберите записи, затем в выпадающем списке действий выберите «Экспортировать выбранные ...» и нажмите 
«Выполнить». Файл CSV скачается автоматически.

## 12. Советы

- Для быстрого поиска используйте строку поиска вверху списка записей.
- Фильтры справа позволяют отсеять записи по статусу, дате, менеджеру и т.п.
- Поля с типом `date` или `datetime` можно выбирать через календарь.
- При создании контракта не забудьте указать **менеджера**, иначе уведомления не будут отправляться.

Если вам нужны дополнительные инструкции по конкретным разделам, обратитесь к документации проекта или задайте вопрос.
