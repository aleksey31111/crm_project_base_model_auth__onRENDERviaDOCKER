### 🚀 Краткий обзор и возможности

Данный Telegram-бот интегрирован с Django-бэкендом для уведомлений о ключевых событиях и управления платежами.

**Основные возможности:**

*   **Уведомления CRM:** автоматически отправляет важные сообщения в личный кабинет пользователя в Telegram о событиях в CRM [7†L5-L10].
*   **Команда `/payments`:** показывает список последних платежей по контрактам, включая только что добавленные.
*   **Команда `/pay <id_контракта>`:** генерирует индивидуальную ссылку на оплату, которую можно сразу переслать клиенту.
*   **Идентификация пользователей:** автоматически распознаёт сотрудников по Telegram ID, связанному с их профилем.

Технически бот работает через Webhook — это способ, при котором Telegram сам отправляет обновления (сообщения, команды) на ваш сервер, что обеспечивает мгновенную реакцию и меньше нагружает систему [2†L7-L8].

### 🤖 Подготовка к работе с Telegram

Для создания нового бота и получения токена обратитесь к официальному боту **@BotFather** [6†L26-L34]:

1.  В поиске Telegram найдите `@BotFather`.
2.  Отправьте ему команду `/newbot`.
3.  Следуйте инструкциям: придумайте **имя** (отображается пользователям) и **username** (должен заканчиваться на "bot", например, `MyAwesomeCrmBot`).
4.  После успешного создания вы увидите **токен** (например, `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11`).
5.  **Важно:** сохраните этот токен в надёжном месте — это ключ доступа к вашему боту.

### ⚙️ Установка и настройка

1.  **Установите зависимости:**
    В терминале вашего проекта (в папке `crm_project_base_model_auth`) выполните команду:
    ```bash
    pip install python-telegram-bot
    ```

2.  **Добавьте токен в переменные окружения:**
    Чтобы не хранить токен в коде, добавьте его в файл `.env` в корне проекта [6†L35-L41]:
    ```
    TELEGRAM_BOT_TOKEN=ваш_скопированный_токен
    ```

3.  **Подключите приложение `telegram_bot` в настройках Django:**
    *   Откройте файл `crm_project_base_model_auth/settings.py`.
    *   Найдите список `INSTALLED_APPS` и добавьте туда `'telegram_bot'` (обычно в конец списка).

4.  **Выполните миграции:**
    Эта команда создаст необходимые таблицы в базе данных для хранения данных о пользователях бота.
    ```bash
    python manage.py migrate
    ```

### 🔗 Настройка Webhook

Webhook — это автоматический канал, по которому Telegram будет отправлять команды и сообщения вашему боту. Без этого бот не будет реагировать [2†L6-L8][11†L7-L8].

*   **URL для Webhook:** `https://ваш_домен/telegram/webhook/`
    При локальной разработке можно использовать инструменты для проброса локального сервера в интернет (например, `ngrok`).

**Установка Webhook через браузер:**

1.  Скопируйте ваш токен (из `.env`).
2.  В адресной строке браузера соберите и откройте следующий URL, заменив `<YOUR_BOT_TOKEN>` и `<YOUR_DOMAIN>`:
    `https://api.telegram.org/bot8289853470:AAFwQKmmicCm2eT3U6qezLVtV853yBt5Qlo/setWebhook?url=https://affection-paltry-humorless.ngrok-free.dev/telegram/webhook/`

**Подробная проверка:**

Для более детальной проверки, вы можете использовать команду `curl`. Откройте терминал и выполните:
```bash
curl -F "url=https://<YOUR_DOMAIN>/telegram/webhook/" https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook
```

**Пример успешного ответа:**
```json
{"ok":true,"result":true,"description":"Webhook is already set"}
```

Если что-то пошло не так, вы можете снять Webhook командой:
```bash
curl https://api.telegram.org/bot<YOUR_BOT_TOKEN>/deleteWebhook
```

### ▶️ Запуск и использование

Бот работает автоматически, как часть вашего Django-проекта, вам не нужно запускать его отдельной командой.

#### Активация бота для сотрудника

Чтобы бот начал отправлять уведомления конкретному сотруднику, необходимо связать его Telegram-профиль с учётной записью в CRM.

1.  **Сообщите сотруднику Telegram ID своего бота.** Он называется `username` (тот, что вы создали в BotFather, например `@MyAwesomeCrmBot`).
2.  Пользователь ищет в Telegram этого бота и нажимает **"Старт"**. В ответ бот пришлёт его уникальный Telegram ID.
3.  **Администратор** системы входит в панель администратора Django (`/admin`), находит профиль этого сотрудника (в `телеграм-бот/пользователи-бота` или в настройках пользователя) и вводит полученный ID.
4.  **Связь готова.** Теперь бот сможет отправлять этому сотруднику персональные уведомления.

### 💬 Доступные команды

Команды отправляются боту в личные сообщения.

*   **`/start`** — Приветственное сообщение. Начать диалог с ботом всегда следует с этой команды. Бот предложит список доступных действий и может запросить ваш Telegram ID, если его нет в системе.
*   **`/payments`** — Получить список последних платежей. Бот покажет информацию о новых добавленных платежах (сумма, контракт, дата), которая вам важна.
*   **`/pay <id_контракта>`** — Получить ссылку на оплату по контракту. Например, боту отправляется команда `/pay 7`, и он создаёт уникальную ссылку для оплаты по контракту с ID 7.

### 🐛 Устранение неполадок

*   **Бот не отвечает на команды.** Вероятно, проблема в Webhook. Проверьте правильность настройки адреса и то, что ваш Django-сервер доступен из интернета.
*   **Уведомления не приходят.** Убедитесь, что Telegram ID пользователя в системе верен, и что в настройках уведомлений у пользователя в CRM включены соответствующие опции.
*   **Ошибка `NoReverseMatch`.** Если бот пытается отправить ссылку, которая не создаётся, проверьте правильность имени URL-маршрута в файле `contracts/urls.py`. Для создания платежей используется маршрут с именем `fake_payment_page`.

### 🔐 Приложение: настройка переменных окружения (.env)

Файл `.env` находится в корневой папке вашего проекта (`crm_project_base_model_auth`). Вот список важных переменных и пример их заполнения:

```
# Django
SECRET_KEY=your_secret_key
DEBUG=True

# Database (если используете PostgreSQL)
DB_HOST=localhost
DB_NAME=your_db_name
DB_USER=your_db_user
DB_PASSWORD=your_db_password
DB_PORT=5432

# Telegram Bot
TELEGRAM_BOT_TOKEN=<TELEGRAM_BOT_TOKEN>
```

Сохраните файл и перезапустите сервер для применения изменений.

Если что-то пойдёт не так, сверьтесь с разделом «Устранение неполадок» или проверьте логи.
