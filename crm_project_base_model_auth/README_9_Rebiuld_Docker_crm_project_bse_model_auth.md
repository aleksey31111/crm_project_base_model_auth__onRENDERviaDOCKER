## Полная пересборка Docker-окружения для `crm_project_base_model_auth` с нуля

Вы пересобрали проект (возможно, изменили код) и хотите полностью очистить старые контейнеры, тома и пересоздать всё заново.  
Ниже – пошаговая инструкция для **Windows (CMD/PowerShell)**, которая удалит старую базу данных, пересоберёт образы, выполнит миграции и зальёт свежие тестовые данные.

---

### 1. Остановите и удалите все контейнеры, сети и тома (полная очистка)

```bash
docker-compose down -v
```

- `-v` удаляет **именованные тома** (включая `postgres_data`, где хранится БД).  
  После этого база данных будет полностью уничтожена.

Если хотите также удалить образы (чтобы пересобрать их «с нуля» без кэша):

```bash
docker-compose down -v --rmi all
```

Но обычно достаточно `-v`.

---

### 2. Пересоберите образы и запустите контейнеры

```bash
docker-compose up -d --build
```

- `--build` пересобирает образы, даже если они уже были.
- `-d` запускает в фоновом режиме.

Если вы меняли только код (без зависимостей), можно не пересобирать, но для гарантии оставляем `--build`.

---

### 3. Выполните миграции (создание таблиц)

```bash
docker-compose exec web python manage.py migrate
```

Применит все миграции в чистой базе данных.

---

### 4. Залейте тестовые данные (специальный скрипт для Docker)

```bash
docker-compose exec web python manage.py populate_db_docker
```

Этот скрипт создаст:
- Пользователей (`admin`/`admin123`, менеджеров, продажников и т.д.)
- Клиентов (юр. лица, ИП, физ. лица)
- Контракты (разные статусы оплаты)
- Платежи (с фейковыми `yookassa_id`)
- Задачи (включая просроченные, с назначенными исполнителями)
- Комментарии, отчёты, уведомления

> **Примечание:** Скрипт безопасен для многократного запуска (использует `get_or_create`), но при полной пересборке база пуста – все записи создадутся заново.

---

### 5. Соберите статику (CSS, JS, изображения админки)

```bash
docker-compose exec web python manage.py collectstatic --noinput
```

---

### 6. Проверьте работоспособность

- Откройте админку: `http://localhost/admin`  
  Логин: `admin`  
  Пароль: `admin123`
- Проверьте наличие клиентов, контрактов, задач.
- Откройте главную страницу (дашборд) – графики и карточки должны отображать данные.

---

### 7. (Опционально) Перезапустите Celery (если необходимо)

Ваш `docker-compose.yml` уже запускает `celery` и `celery-beat`.  
Если вы меняли задачи, можно перезапустить только эти сервисы:

```bash
docker-compose restart celery celery-beat
```

---

## Краткий чек-лист (одной командой)

Если вы хотите выполнить всё одной серией команд (для PowerShell):

```powershell
docker-compose down -v
docker-compose up -d --build
docker-compose exec web python manage.py migrate
docker-compose exec web python manage.py populate_db_docker
docker-compose exec web python manage.py collectstatic --noinput
```

После этого ваша CRM полностью готова к работе.

---

## Почему именно `populate_db_docker`, а не `populate_db`?

- `populate_db.py` рассчитан на локальный запуск (SQLite) и может использовать абсолютные пути, которые не подходят для Docker.
- `populate_db_docker.py` адаптирован для среды внутри контейнера (использует `get_or_create`, генерирует фейковые `yookassa_id` и `payment_url`, учитывает, что реальной ЮKassa нет).

**Если вы изменили модели или добавили новые поля**, возможно, скрипт потребуется обновить – но для базового набора данных он полностью работоспособен.

---

## Что делать, если после пересборки что-то не работает?

1. Проверьте логи контейнера `web`:
   ```bash
   docker-compose logs web
   ```
2. Проверьте, что переменные окружения в `.env` корректны (особенно `DEBUG`, `DB_HOST=db`).
3. Убедитесь, что порт `8000` не занят другими приложениями.
4. Попробуйте пересобрать без кэша: `docker-compose build --no-cache`
