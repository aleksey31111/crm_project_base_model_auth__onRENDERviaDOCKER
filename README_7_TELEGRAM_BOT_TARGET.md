## 🚀 Полное руководство по использованию Telegram-бота для CRM (crm_project_base_model_auth)

Бот интегрирован в проект и работает в режиме **long polling** (постоянный опрос). Он отправляет уведомления о задачах, контрактах и платежах, поддерживает команды `/payments`, `/pay` и `/bind`.

### 🤖 Возможности бота

- **Автоматические уведомления** о новых задачах, контрактах и успешных платежах (через сигналы Django).
- **Команда `/payments`** – последние 5 платежей по контрактам, где пользователь является менеджером.
- **Команда `/pay <id_контракта>`** – получить инлайн-кнопку со ссылкой на тестовую оплату.
- **Команда `/bind <username>`** – привязать Telegram-аккаунт к учётной записи CRM.

### 🛠 Подготовка и установка

1. **Создайте бота у @BotFather** в Telegram, получите токен.
2. Убедитесь, что проект работает и установлены зависимости:
   ```bash
   pip install python-telegram-bot requests
   ```
3. Приложение `telegram_bot` уже добавлено в `INSTALLED_APPS` (проверьте `settings.py`).
4. Выполните миграции:
   ```bash
   python manage.py migrate
   ```

### 🔐 Настройка переменных окружения

В корне проекта (рядом с `manage.py`) в файле `.env` **обязательно** пропишите:

```
TELEGRAM_BOT_TOKEN=ваш_токен_от_BotFather
BASE_URL=https://ваш_домен   # для продакшена, для локальной разработки можно http://127.0.0.1:8000
```

В `settings.py` должны быть:
```python
TELEGRAM_BOT_TOKEN = os.getenv('TELEGRAM_BOT_TOKEN')
BASE_URL = os.getenv('BASE_URL', 'http://127.0.0.1:8000')
```

### ▶️ Запуск бота

Бот работает через **Django management command**. В вашем приложении есть файл `management/commands/runbot.py` (вы его выложили как `runbot.py` – убедитесь, что он лежит по пути `telegram_bot/management/commands/runbot.py`).

Запустите бота командой:

```bash
python manage.py runbot
```

Вы увидите сообщение: `Бот запущен`. Бот начнёт опрашивать Telegram и отвечать на команды. **Бот должен работать постоянно** – в продакшене используйте systemd или supervisor.

> ⚠️ Если файл `runbot.py` лежит неправильно, создайте папки: `telegram_bot/management/commands/` и поместите туда `runbot.py`.

### 👤 Привязка аккаунта сотрудника

Чтобы сотрудник получал уведомления и мог использовать команды, нужно связать его Telegram с учётной записью Django.

1. Пользователь находит бота по его **username** (например, `@MyCrmBot`) и отправляет команду:
   ```
   /bind логин_в_crm
   ```
   Например: `/bind ivanov`.

2. Бот проверяет, существует ли пользователь с таким логином. Если да – создаёт запись в модели `TelegramUser`, сохраняя `chat_id` и пользователя.

3. После привязки бот будет отправлять этому пользователю уведомления и обрабатывать его команды.

> **Администратор** может также привязать пользователя вручную через админку Django (`/admin/telegram_bot/telegramuser/`), указав `user` и `chat_id`.

### 💬 Доступные команды

Все команды отправляются боту в личные сообщения.

| Команда | Пример | Что делает |
|---------|--------|-------------|
| `/start` | `/start` | Приветствие и напоминание о `/bind`. |
| `/bind username` | `/bind alex` | Привязывает текущий Telegram-аккаунт к пользователю CRM с логином `alex`. |
| `/payments` | `/payments` | Показывает последние 5 платежей по контрактам, где вы – менеджер. |
| `/pay <id_контракта>` | `/pay 7` | Присылает кнопку «Оплатить» со ссылкой на тестовую страницу оплаты контракта. |

### 📨 Как работают уведомления

В `signals.py` описаны сигналы:

- **Создание задачи** → уведомление исполнителю (`assigned_to`).
- **Создание контракта** → уведомление менеджеру (`manager`).
- **Успешная оплата (`yookassa_status == 'succeeded'`)** → уведомление менеджеру контракта.

Все уведомления отправляются через функцию `send_telegram_notification`, которая вызывает прямые HTTPS-запросы к API Telegram. Для этого используется `requests`, а не асинхронный вызов, чтобы работать внутри синхронных сигналов.

**Что нужно для работы уведомлений:**
- У пользователя в CRM должен быть заполнен `username` (логин).
- Пользователь должен привязать Telegram через `/bind`.
- В модели `TelegramUser` должно быть `is_active = True` (по умолчанию true).

### 🔧 Устранение неполадок

| Проблема | Вероятная причина | Решение |
|----------|-------------------|---------|
| Бот не отвечает на команды | Бот не запущен | Выполните `python manage.py runbot` в отдельном терминале или настройте демона. |
| Ошибка `TELEGRAM_BOT_TOKEN not found` | Токен не задан в `.env` | Добавьте `TELEGRAM_BOT_TOKEN=...` и перезапустите бота. |
| `/bind` пишет «Пользователь не найден» | В CRM нет пользователя с таким логином | Проверьте точность username (поле `username` в модели `User`). |
| Уведомления не приходят | Пользователь не привязан или `is_active=False` | Привяжите через `/bind` или проверьте админку. |
| Команда `/pay` не генерирует ссылку | Неверный ID контракта или контракт не принадлежит пользователю | Убедитесь, что ID правильный и в контракте указан `manager` = текущий пользователь. |
| Ошибка импорта `runbot` | Файл `runbot.py` не в `management/commands` | Создайте структуру: `telegram_bot/management/commands/runbot.py`. |
